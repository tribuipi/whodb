# Radix UI → Base UI Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all 16 `@radix-ui/*` packages with `@base-ui/react` using the shadcn CLI's `base-vega` style, then fix the small set of manual post-migration issues.

**Architecture:** The shadcn CLI regenerates all `frontend/src/components/ui/` files in one pass by switching `components.json` style to `base-vega`. Post-CLI work is confined to two business-layer files that use `asChild` and a final cleanup of radix package removal.

**Tech Stack:** React 18, TypeScript, Tailwind CSS v4, `@base-ui/react` (replacing all `@radix-ui/*`), shadcn CLI (`pnpm dlx shadcn@latest`), pnpm

## Task 3: Fix asChild in sidebar/sidebar.tsx

`asChild` appears 10 times across three patterns. Fix each based on what Task 2 found.

**Files:**
- Modify: `frontend/src/components/sidebar/sidebar.tsx`

### Pattern A — SidebarMenuButton wrapping a `<Link>` (lines 143, 754, 774, 789, 805)

**If `SidebarMenuButton` now has a `render` prop:**

```tsx
// Before
<SidebarMenuButton asChild tooltip={route.title}>
  <Link
    to={route.path}
    className={cn("flex items-center gap-2", {
      "font-bold": pathname === route.path,
    })}
  >
    {route.icon}
    {open && <span>{route.title}</span>}
  </Link>
</SidebarMenuButton>

// After
<SidebarMenuButton
  tooltip={route.title}
  render={
    <Link
      to={route.path}
      className={cn("flex items-center gap-2", {
        "font-bold": pathname === route.path,
      })}
    />
  }
>
  {route.icon}
  {open && <span>{route.title}</span>}
</SidebarMenuButton>
```

Apply the same `render` prop conversion at lines 143, 754, 774, 789, and 805 — each follows this same `asChild` + `<Link>` pattern.

**If `SidebarMenuButton` has NO render prop (renders as button only):** wrap with a plain anchor/Link instead and remove `SidebarMenuButton`:

```tsx
<SidebarMenuItem>
  <Link
    to={route.path}
    className={cn(sidebarMenuButtonVariants(), "flex items-center gap-2", {
      "font-bold": pathname === route.path,
    })}
  >
    {route.icon}
    {open && <span>{route.title}</span>}
  </Link>
</SidebarMenuItem>
```

### Pattern B — SidebarMenuButton wrapping a `<div onClick>` (lines 640, 806)

The intent is a clickable menu item that isn't a link. Remove `asChild` and pass `onClick` directly to `SidebarMenuButton`:

```tsx
// Before
<SidebarMenuButton asChild tooltip={t('logOutProfile')}>
  <div className="flex items-center gap-sm text-nowrap w-fit cursor-pointer" onClick={handleLogoutProfile}>
    <ArrowLeftStartOnRectangleIcon className="w-4 h-4" />
    {open && <span>{t('logOutProfile')}</span>}
  </div>
</SidebarMenuButton>

// After
<SidebarMenuButton tooltip={t('logOutProfile')} onClick={handleLogoutProfile} className="flex items-center gap-sm text-nowrap w-fit cursor-pointer">
  <ArrowLeftStartOnRectangleIcon className="w-4 h-4" />
  {open && <span>{t('logOutProfile')}</span>}
</SidebarMenuButton>
```

### Pattern C — SidebarMenuButton wrapping a `<DropdownMenu>` (lines 646, 811)

This uses `asChild` to avoid a button-in-button DOM error while applying sidebar button styling to the dropdown trigger area. Remove the outer `SidebarMenuButton asChild` wrapper entirely — the `DropdownMenuTrigger` wrapping the `<Button>` is sufficient:

```tsx
// Before
<SidebarMenuButton asChild>
  <DropdownMenu>
    <DropdownMenuTrigger asChild className={cn({ "hidden": !open })}>
      <Button
        className="flex items-center justify-center p-1 rounded hover:bg-gray-100 dark:hover:bg-neutral-800 ml-2"
        aria-label={t('moreLogoutOptions')}
        variant="ghost"
      >
        <ChevronDownIcon className="w-4 h-4" />
      </Button>
    </DropdownMenuTrigger>
    <DropdownMenuContent side="right" align="start">
      ...
    </DropdownMenuContent>
  </DropdownMenu>
</SidebarMenuButton>

// After (remove the outer SidebarMenuButton wrapper)
<DropdownMenu>
  <DropdownMenuTrigger render={<Button
    className={cn("flex items-center justify-center p-1 rounded hover:bg-gray-100 dark:hover:bg-neutral-800 ml-2", { "hidden": !open })}
    aria-label={t('moreLogoutOptions')}
    variant="ghost"
  />}>
    <ChevronDownIcon className="w-4 h-4" />
  </DropdownMenuTrigger>
  <DropdownMenuContent side="right" align="start">
    ...
  </DropdownMenuContent>
</DropdownMenu>
```

### Pattern D — TooltipTrigger asChild (line 842)

```tsx
// Before
<TooltipTrigger asChild>
  <button className={cn(
    "rounded-full p-1 text-muted-foreground/60 hover:text-muted-foreground transition-colors",
    updateInfo?.UpdateInfo?.updateAvailable && "text-blue-500/70 hover:text-blue-400"
  )} data-testid="sidebar-version-info">
    <InformationCircleIcon className="w-4 h-4" />
  </button>
</TooltipTrigger>

// After
<TooltipTrigger render={<button
  className={cn(
    "rounded-full p-1 text-muted-foreground/60 hover:text-muted-foreground transition-colors",
    updateInfo?.UpdateInfo?.updateAvailable && "text-blue-500/70 hover:text-blue-400"
  )}
  data-testid="sidebar-version-info"
/>}>
  <InformationCircleIcon className="w-4 h-4" />
</TooltipTrigger>
```
