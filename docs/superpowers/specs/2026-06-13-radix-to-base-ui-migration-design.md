**Date:** 2026-06-13  
**Branch:** feat/sql-editor-redesign  
**Reference:** https://github.com/shadcn-ui/ui/discussions/9562

## Summary

Replace all 16 `@radix-ui/*` packages with `@base-ui/react` using the shadcn CLI's `base-vega` style. All radix-ui usage is confined to `frontend/src/components/ui/` wrapper files — the CLI regenerates them in one pass. A small set of manual fixes follows.

## Migration Steps

### Step 1 — Update components.json

Change `"style": "default"` to `"style": "base-vega"` in `frontend/components.json`.

### Step 2 — Run shadcn CLI

```bash
cd frontend
pnpm dlx shadcn@latest add --all --overwrite --yes
```

This regenerates all UI components against the base-vega registry, replacing radix-ui imports with `@base-ui/react/*` imports and updating data attribute selectors in className strings.

### Step 3 — Fix asChild usage

`asChild` does not exist in base-ui. After the CLI run, grep for remaining `asChild` occurrences:

```bash
grep -rn "asChild" frontend/src --include="*.tsx"
```

Expected remaining sites (not in UI primitives):

- `frontend/src/components/sidebar/sidebar.tsx` — `SidebarMenuButton asChild` wrapping `<Link>` elements, and `DropdownMenuTrigger asChild`
- `frontend/src/components/ui/copy-button.tsx` — `TooltipTrigger asChild`

**Before fixing `sidebar/sidebar.tsx`**, inspect the regenerated `ui/sidebar.tsx` to understand what API `SidebarMenuButton` now exposes. If it accepts a `render` prop, convert `asChild` + child element to `render={<Link to="..." />}`. If the regenerated component removed `asChild` in a different way (e.g., accepts children directly), follow that pattern instead.

For base-ui primitives that do accept `render`, the pattern is:

```tsx
// Before (radix)
<TooltipTrigger asChild>
  <Button />
</TooltipTrigger>

// After (base-ui)
<TooltipTrigger render={<Button />} />
```

### Step 4 — Verify CSS data attribute selectors

The CLI rewrites selectors inside the UI component files. Scan remaining files for stale radix-specific patterns and update:

| Old (radix) | New (base-ui) |
|---|---|
| `data-[state=open]:` | `data-open:` |
| `data-[state=closed]:` | `data-closed:` |
| `data-[state=active]:` | `data-active:` |
| `data-[state=checked]:` | `data-checked:` |
| `data-[state=unchecked]:` | `data-unchecked:` |
| `origin-[--radix-*-transform-origin]` | `origin-(--transform-origin)` |

### Step 5 — Remove radix packages

```bash
cd frontend
pnpm remove @radix-ui/react-accordion @radix-ui/react-alert-dialog \
  @radix-ui/react-checkbox @radix-ui/react-context-menu @radix-ui/react-dialog \
  @radix-ui/react-dropdown-menu @radix-ui/react-label @radix-ui/react-popover \
  @radix-ui/react-scroll-area @radix-ui/react-select @radix-ui/react-separator \
  @radix-ui/react-slot @radix-ui/react-switch @radix-ui/react-tabs \
  @radix-ui/react-tooltip @radix-ui/react-visually-hidden
```

### Step 6 — Type check & build

```bash
cd frontend && pnpm run build:ce
```

Fix any remaining type errors (Checkbox `checked` must be `boolean`, not `boolean | "indeterminate"`).

## Known Breaking Changes

| Change | Location | Fix |
|---|---|---|
| `asChild` removed | `sidebar.tsx`, `copy-button.tsx` | Use `render` prop |
| `Checkbox checked` is `boolean` only | Any checkbox consumer passing `"indeterminate"` | Use `indeterminate` prop instead |
| `data-[state=*]` selectors | UI component classNames (handled by CLI) | CLI rewrites these |
| `--radix-*-transform-origin` CSS var | `tooltip.tsx`, `popover.tsx` (handled by CLI) | CLI rewrites these |
| `DialogPrimitive.Overlay` → `Backdrop` | `dialog.tsx` (handled by CLI) | CLI regenerates |
| `DialogPrimitive.Content` → `Popup` | `dialog.tsx` (handled by CLI) | CLI regenerates |

---

## Success Criteria

1. `pnpm run build:ce` passes with no type errors.
2. No `@radix-ui/*` imports remain anywhere in `frontend/src/`.
3. No `asChild` props remain anywhere in `frontend/src/`.
4. `frontend/package.json` contains no `@radix-ui/*` dependencies.
5. App renders correctly: sidebar navigation, dialogs, tooltips, dropdowns, tabs, checkboxes, switches, selects all function.
